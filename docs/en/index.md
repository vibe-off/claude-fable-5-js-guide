## Roadmap

The guide is organized in three progressive tiers. Work through them in order:

| Stage | Content | Goal |
| --- | --- | --- |
| Fundamentals | Syntax, functions, arrays & objects, DOM, events | Build interactive web pages on your own |
| Key Concepts | Scope, closures, this, prototypes, async, fetch | Read other people's (and AI's) code; handle interviews |
| Beyond the Basics | Tooling, storage, common techniques, AI collaboration | Entry-level readiness for real projects; a bridge to Vue 3 |

## How to Use This Guide

1. **Type out every code sample yourself.** Understanding it is not the same as being able to write it — muscle memory matters.
2. **Finish each section's exercises before moving on.** Every exercise ships with a collapsible reference answer: attempt first, compare after.
3. **Use AI well, but don't let it learn for you.** Having AI explain code, help you debug, or quiz you is great; copy-pasting AI answers into assignments only hurts you — interviewers will ask you to modify code on the spot. See [Learning with AI](/en/advanced/learning-with-ai).
4. **Look up every word you don't know.** MDN (developer.mozilla.org) is the authoritative frontend reference — make consulting docs a habit.

## By the End

After finishing this guide, you should be able to:

- Independently build a static project with form validation, list rendering, and API requests;
- Read and modify unfamiliar JS code, and explain what it does;
- Answer classic interview questions: What is a closure? What does `this` point to? What problem do Promises solve?
- Enter [Vue 3](/en/advanced/next-vue) with the right knowledge structure in place.
